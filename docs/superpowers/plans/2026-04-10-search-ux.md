> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Address ChromaDB's `$ne` operator edge cases for missing keys, and improve UX by tagging archived results when querying with `--deep`.

**Architecture:** 
1. `mempalace/searcher.py`: Modify the `where` clause logic. Instead of relying on `{"wing": {"$ne": "archive"}}` (which can drop records without a wing key), we should construct the query to avoid this ChromaDB quirk, or accept that we ensure all our records have wings. Since we normalize all new inserts, but old DBs might lack keys, the safest approach in Chroma is often to do the filtering post-query if we can't guarantee metadata shape, or to use an `$or` clause if Chroma supports it, but post-filtering is 100% reliable for this edge case. However, let's look at `mempalace.normalize` - we might already enforce `wing`. To be safe and fast, we'll post-filter the results if `deep=False`.
2. `mempalace/cli.py`: Modify `cmd_search` to prepend `[archive]` to the output if the metadata shows `wing == "archive"`.

**Tech Stack:** Python 3, ChromaDB.

    query: str,

    room: str = None,
    n_results: int = 5,
    deep: bool = False,
) -> list[dict]:
    # ...
    where_clauses = []
    if wing:
        where_clauses.append({"wing": wing})
    if room:
        where_clauses.append({"room": room})

    where = None
    if len(where_clauses) == 1:
        where = where_clauses[0]
    elif len(where_clauses) > 1:
        where = {"$and": where_clauses}

    try:
        # Request more results if we need to post-filter
        fetch_n = n_results * 2 if not deep and not wing else n_results
        
        results = col.query(
            query_texts=[query],
            n_results=fetch_n,
            where=where,
            include=["documents", "metadatas", "distances"],
        )

        for j, doc in enumerate(docs):

                "document": doc,
                "metadata": meta,
                "distance": dists[j] if j < len(dists) else 0.0,
            })
            
            if len(filtered_docs) >= n_results:
